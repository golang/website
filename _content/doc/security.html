策略实施

报告安全漏洞

如您发现任何问题，请向我们报告。 本文档描述了具体报告流程和答复信息。

Go 发行版中的所有安全漏洞均应通过邮件发送至security@golang.org，该邮件会传送至某小型安全团队。 您的邮件将在24小时内得到确认，并在72小时内收到更详细的回复，在其中会指明处理漏洞的后续流程。

为防止您的报告被当作垃圾邮件，请在邮件中任意处添加“vulnerability”一词，同时，邮件请使用描述性标题。

在对您的报告进行初次答复后，安全团队将尽可能地确保您知晓有关漏洞修复及完整公告的进展情况。这些更新信息将至少每五天发送一次。实际上，更有可能是每24-48小时就发送一次。

若您在48小时内未收到回复，或者在过去五天内没有收到安全团队的通知，请直接与Go安全团队联系：

  • 首席安全协调员： Filippo Valsorda。
  • 副安全协调员： Adam Langley。
  • 若您未收到回复，请向golang-dev@googlegroups.com发送邮件，或者使用golang-dev Web页面。

请注意，golang-dev 是一个公开论坛，请不要透露具体信息，只需声明您正在尝试联系安全团队成员即可。

将现有 issues 标记为与安全相关

若您确定某个现存的 issue 有安全漏洞，请发送邮件至security@golang.org。邮件需要包含 issue 号和简短的说明，说明为什么要根据安全策略进行处理。

安全相关问题的披露流程

Go 使用以下披漏流程：

  1. 收到漏洞报告后，为其分配一个主理人，他负责协调漏洞修复和发布。
  2. 确认该漏洞，并列出受其影响的软件。
  3. 审核代码以找出任何类似的潜在漏洞。
  4. 与代码提交人员协商后，若需要一个 CVE-ID 号，则主理人向 oss-distros 发送邮件获取。
  5. 为最近的两个主要发行版和 head/master 版本准备了修复程序，但这些补丁尚未提交到公共仓库。
  6. 向golang-announce邮件列表发送通知，以使下游用户有时间对其系统进行更新。
  7. 此通知三个工作日后，补丁将提交到公共仓库，并发布新的 Go 版本。
  8. 在应用补丁当日，发送通知到 golang-announce， golang-dev，和 golang-nuts。

整个过程可能需要一些时间，尤其是在协调过程涉及到其他项目维护人员时。我们将尽一切努力尽快修复该漏洞，但请务必遵循上述流程，以确保披漏流程的一致。

对于安全问题，包括 CVE-ID 号的分配，具体情况列于 CVEDetails 网站上的 "Golang" 项目及 国家漏洞公开网站 。

接收安全更新

最高效获取安全通知的方法是订阅golang-announce邮件列表，任何涉及安全问题的信息都将带有[security]前缀。

意见和建议

如果您有任何改进此策率的意见和建议，请发送电子邮件至golang-dev@golang.org。

用于向security@golang.org发送邮件的 PGP 密钥

我们接受 PGP 加密电子邮件，但大多数安全团队并不常用 PGP，这多少有些不便，因此请仅将 PGP 用于重要的安全报告邮件。
